# Phase 07 Plan — notebook-lm-pi Integration

## Goal
Integrate notebook-lm-pi for free AI RAG processing and deliverable generation (infographics, slide decks, podcasts, audio overviews, mind maps, flashcards). Defer API key/auth to end.

## Steps

### 07-01: notebook-lm-pi Handler (Stub + Real)
- [ ] Add `notebook-lm-pi` to requirements.txt
- [ ] Create `src/agent/handlers/notebook_lm_handler.py`
- [ ] `NotebookLMHandler` class with:
  - `is_authenticated()` — check if Google auth tokens exist
  - `upload_source(url)` — upload YouTube/URL/article to Notebook LM
  - `query(question)` — ask question about uploaded sources
  - `generate_deliverable(deliverable_type)` — create infographic/slides/podcast/mind-map/flashcards
- [ ] Add `_NOTEBOOK_LM_NOT_CONFIGURED` flag when API key/auth missing
- [ ] Stub methods return helpful "not configured yet" messages pointing to Legion setup

### 07-02: Deliverable Generation Flow
- [ ] Create `src/agent/deliverables.py`
- [ ] `format_deliverable_options(results)` — offer "Generate podcast" / "Generate infographic" / etc.
- [ ] After Tavily research results, offer deliverable options
- [ ] When user selects, call notebook-lm-pi (or queue if not configured)

### 07-03: Integration with Research Pipeline
- [ ] Extend `research_handler.py` to offer deliverable generation after results
- [ ] Add deliverable type selection to research flow
- [ ] Return deliverable links/files to user when ready

### 07-04: Tests
- [ ] `tests/test_notebook_lm_handler.py` — test stub responses (no API needed)
- [ ] `tests/test_deliverables.py` — test offer formatting

## Success Criteria
- notebook-lm-pi handler returns helpful "not configured" message without crashing
- When configured, can upload sources and generate deliverables
- Research results offer deliverable generation as next step
- API keys deferred to Legion setup (end of v2.0)
