# Phase 08 Plan — Research Session Logging

## Goal
Track research history in SQLite. Each research session logged with timestamp, query, results, and deliverables so you can see your research history over time.

## Steps

### 08-01: Database Schema
- [ ] Add `research_sessions` table to `Database` class
- [ ] Schema: `id, timestamp, query, seed_content_id, results_count, deliverable_types, notes`
- [ ] Add `insert_research_session()` method
- [ ] Add `get_recent_sessions(limit=10)` method
- [ ] Add `get_session_history(topic_filter)` method

### 08-02: Session Logging in Research Handler
- [ ] In `research_handler.py`, log session start before Tavily search
- [ ] Log session end after results returned
- [ ] Include: query, seed_content_id, results_count, deliverable types
- [ ] Add session ID to result metadata for reference

### 08-03: Session History Display
- [ ] In `research_utils.py`, add `format_session_history(sessions)`
- [ ] Display: date, query, results count, deliverables generated
- [ ] Add trigger: "what have I researched?" or "show my research history"
- [ ] Integrate into surfacing handler

### 08-04: Tests
- [ ] `tests/test_session_logging.py` — test insert + query methods

## Success Criteria
- Research sessions automatically logged in SQLite
- User can ask "what have I researched?" and see history
- Session history shows query, date, results count, deliverables
